Claude Code is powerful, and like any powerful tool it rewards good habits and punishes careless ones. The good news is that the habits that keep you safe are simple and quickly become second nature. None of them require any technical skill. They are mostly about pace, attention, and knowing where your data belongs.

## Six habits worth building

- **Take small steps.** Small changes are easy to review and easy to undo. A tool moving in small steps rarely surprises you.
- **One task at a time.** Ask for a single clear thing, see it through, then ask for the next. Bundling five requests into one message makes the result harder to check and harder to trust.
- **Verify everything.** Read what Claude proposes, and check any figure, date, or name against the source. Speed is only useful if the output is right.
- **Keep confidential data where it belongs.** Practice on samples, and never move restricted material outside the boundaries your firm's policy allows.
- **Use Git so you can undo.** Git (covered elsewhere in this track) records your work so you can always return to an earlier state. It is your safety net.
- **Involve IT when unsure.** If you are uncertain about permissions, data handling, or setup, ask your IT or security team before proceeding. A short question now prevents a real problem later.

## Risky habits and better ones

Most trouble comes from a small number of tempting shortcuts. Here is each one alongside the habit that replaces it.

<ComparisonTable title="Risky habit versus better habit" caption="The better habit is rarely slower. It just removes the chance of a costly surprise.">

| Risky habit | Better habit |
| --- | --- |
| Approving a change without reading it | Reading the plan and the exact change first, then approving |
| Trying it on live deal data straight away | Testing on a throwaway or sample folder until the loop feels familiar |
| Sending one huge, vague request | Asking for one small, clear task at a time |
| Trusting figures because they look right | Checking every figure, date, and name against the source |
| Working with no way back | Using Git so any change can be undone |
| Guessing at a permissions or data question | Asking IT or security before you proceed |

</ComparisonTable>

## Where your data belongs

Good habits around data are the ones that matter most, because a mistake here can be hard to reverse.

<Callout type="confidential" title="Keep restricted data in bounds">
Your files stay on your machine, and only your prompt and the context a task needs are sent to Claude. That is a strong default, but it does not replace your firm's rules. Never move restricted or confidential material beyond what your firm's policy allows, whether into a new folder, a shared location, or any tool that has not been approved for it. When in doubt about a specific document, treat it as restricted and check first.
</Callout>

## You own the output

The final habit is a mindset. Claude Code is an assistant, not an authority. It is fast and often right, but it can also be confidently wrong, and it cannot be held responsible for what you produce. You can.

<Callout type="warning" title="You are accountable">
Claude can make mistakes, including mistakes that read as perfectly plausible. Whatever you use or share carries your name, not Claude's. Verify the work before it goes anywhere: check the figures against the source, read the wording as your own, and confirm nothing confidential has slipped in. Treat every output as a draft you are responsible for, because that is exactly what it is.
</Callout>

## The takeaway

Safe use of Claude Code comes down to a handful of habits: take small steps, do one task at a time, verify everything, keep confidential data in bounds, use Git so you can undo, and involve IT when you are unsure. Above all, remember that you own the output. Build these habits early and they become invisible, leaving you free to move quickly with the confidence that you are still firmly in control.
