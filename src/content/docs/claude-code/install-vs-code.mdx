Before Claude Code has somewhere to live, you need VS Code. It is a free editor made by Microsoft, and it is one of the most widely used pieces of software in the world. The name looks technical, and it is often used by developers, but you do not need to know how to code to use it. For our purposes it is simply the window that Claude Code runs inside, and a comfortable place to look at your files.

<Callout type="warning" title="Involve IT first">
Use your KPMG-managed device and follow your IT policy for installing software. If you are unsure whether you are allowed to install VS Code, or how, ask IT before you begin. Do not install it on a personal device for any confidential or client work.
</Callout>

## Installing it, step by step

<Steps>
  <Step title="Download VS Code">
    Go to the official Visual Studio Code website and download the version for your operating system. It usually detects whether you are on Windows or Mac and offers the right one. If your firm provides software through a managed portal or company store, use that instead, and check with IT if you are not sure which route applies to you.
  </Step>
  <Step title="Install it">
    Open the file you downloaded and follow the prompts. You can accept the standard options; there is nothing you need to change. Installation takes a minute or two.
  </Step>
  <Step title="Open it">
    Launch VS Code the same way you open any other program. The first time, it may ask a few setup questions about appearance and preferences. You can accept the defaults and move on; none of them affect the work you will do.
  </Step>
  <Step title="Take a quick look around">
    Spend a moment getting your bearings. There are three parts worth knowing:

    - The **file explorer** on the left. This is where your files and folders appear, like a familiar folder view. You will open your deal folder here.
    - The **editor** in the middle. This is the large central area where a file's contents are shown when you open it.
    - The **terminal** at the bottom. This is a text area where Claude Code does its talking. When you type an instruction and read Claude's plan, this is where it happens. It may be hidden at first; you can open it from the View menu.
  </Step>
</Steps>

That is the whole setup for VS Code. You do not need to learn its many features. Knowing where the file explorer, the editor, and the terminal are is enough to get started.

<Callout type="note" title="This is just the container">
Installing VS Code does not install Claude Code. VS Code is the room; Claude Code is what you invite into it. Adding Claude Code, and signing in with your KPMG account, is the next page: [Installing & Signing In to Claude Code](/claude-code/sign-in).
</Callout>
