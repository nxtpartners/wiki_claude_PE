Claude.ai is deliberately simple. There is no dashboard to learn and no settings to wade through before you can start. This page gives you a fast orientation so the screen feels familiar the moment you open it.

The illustrations below recreate the real interface. Your own screen may differ in small ways as Claude updates, but the parts that matter stay in the same place.

## The home screen

When you sign in you land on a new chat. A slim menu sits on the left, and the message box sits in the middle. That is the whole app.

**The left menu** is how you move around:

- **New chat** starts a fresh conversation. Each chat is separate, so use a new one for each task.
- **Chats** lists your recent conversations so you can return to anything.
- **Projects** are dedicated workspaces that hold instructions and a set of files. This is where most serious deal work belongs, covered in the [Projects](/projects/what-projects-are) section.
- **Artifacts** collects the standalone documents Claude builds for you, such as a memo or a model output.

<Callout type="note" title="You are in a KPMG workspace">
On Claude Enterprise and Team the top of the menu shows your organisation, here labelled KPMG. That workspace is what keeps your prompts and uploads out of model training, and it is why these plans are appropriate for confidential work.
</Callout>

## The message box, up close

Almost everything happens in the message box. You type a request, attach files, and press send. The controls are worth knowing by name.

<ClaudeWindow caption="The message box. Attach is the plus on the left; send is the arrow on the right.">

    <ClaudeComposer
      value="Summarise the attached CIM for an initial screen."

      effort="High"
    />
    <Hotspot n={1} x="14%" y="80%" />
    <Hotspot n={2} x="20%" y="10%" label="Attach menu" />
    <Hotspot n={3} x="76%" y="80%" />
    <Hotspot n={4} x="93%" y="80%" />
  </div>
</ClaudeWindow>

1. **The plus button** opens the attach menu. This is where you add a PDF, Word document, or Excel file, or pull a file from a connected source such as Google Drive.
2. **The attach menu** lists where files can come from. Uploading from your computer is the everyday route.
3. **The model selector** shows which Claude model is answering and how hard it is thinking. The default is the right choice for almost all PE work.
4. **The send arrow** submits your message. You can also press Enter.

<Callout type="tip" title="One file or many">
You can attach several files to a single message. Claude reads all of them together, which is how you ask one question across a whole set of documents.
</Callout>

## A conversation

Once you send, the chat fills in. Your messages sit on the right; Claude's replies run full width on the left. Everything you say stays in view, so each new message can build on the last.

To keep going, you simply type again: "Put the financials in a table," or "What should I ask management first?" There is no need to re-attach the file or repeat yourself.

## Where deal work lives: Projects

For anything beyond a quick question, you will work inside a **Project**. A Project pairs a set of standing instructions with a knowledge base of files, so Claude already knows the context every time you open it.

Projects are the heart of using Claude well, and they are how you keep each deal's data separate and confidential. The whole [Projects](/projects/what-projects-are) section walks through setting them up.

## What to take from this

The interface is small on purpose: a menu on the left, a message box in the middle, and Projects for anything that deserves its own workspace. With the layout familiar, the next step is learning to brief Claude properly, which begins with the [Anatomy of a Good Prompt](/foundations/anatomy-of-a-prompt).
