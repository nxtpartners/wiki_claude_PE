# Context — Corrections & Project Rules

Format: `[YYYY-MM-DD] | what went wrong | rule to prevent it`

- [2026-06-30] | Project kickoff | Audience is non-technical PE professionals at KPMG — never let the site feel like developer docs (no Docusaurus look). Polished, editorial, institutional.
- [2026-06-30] | Branding | Use KPMG palette (deep blue #00338D); do NOT use NXT Partners colors or fonts. No logo yet.
- [2026-06-30] | Scope | Primary track is the Claude.ai web app (not the API). Assume Enterprise/Team accounts.
- [2026-07-09] | Scope update | Claude Code is now IN scope as a second track (its own "Claude Code" nav section and glossary terms). The wiki now teaches two tracks: the Claude.ai web app (primary) and Claude Code (secondary). The API remains out of scope.
- [2026-06-30] | Typography decision | Use Inter (sans-only) throughout — clean KPMG-aligned look, NOT serif. KPMG deep blue #00338D primary.
- [2026-06-30] | Dependency pinning | astro-pagefind@2 only supports astro ≤6 — pin astro@6 + @astrojs/mdx@6 (NOT astro 7 / mdx 7) or npm install fails with peer-dep conflict.
- [2026-06-30] | Pagefind asset loading | Never ESM-`import()` `/pagefind/pagefind-ui.js` — Rollup tries to resolve it and the build fails (asset only exists post-build). Inject it as a runtime `<script>` tag instead.
- [2026-06-30] | MDX components in content | Plain `.md` cannot use components like `<Callout>`. Any content page using components MUST be `.mdx` with explicit relative imports.
[2026-06-30] | Style rule | NEVER use em dashes or en dashes anywhere (content, UI text, or code comments). Use commas, colons, parentheses, or split sentences instead.
- [2026-06-30] | Audience wording | NEVER call the audience "deal team(s)". It is KPMG **Private Equity** (broader than deal teams). Use "private equity teams" / "private equity professionals".
- [2026-06-30] | Claude.ai mockups | Any illustration of the Claude.ai interface MUST look like the real product: warm cream background (#FAF9F5 / #F0EEE6), Claude clay/coral accent (#BE5D3A), neutral message bubbles. NEVER use KPMG blue for the chat UI — it makes users think they are on the wrong page or doing something wrong (audience has no AI experience).
- [2026-06-30] | Hero copy tone | Keep landing copy professional and education-first. NO salesy/marketing lines (e.g. "fastest analyst on your deal team"). This is a guide, not an ad.
- [2026-06-30] | YAML frontmatter | Any `description:` containing a colon MUST be wrapped in double quotes or the Astro build fails ("bad indentation of a mapping entry"). After authoring, grep `^description: [^"'].*: ` and quote offenders.
- [2026-06-30] | MDX import depth | `src/content/docs/<section>/*.mdx` imports components THREE levels up (`../../../components/`); root-level docs (`welcome.mdx`, `first-15-minutes.mdx`) use TWO levels (`../../components/`).
- [2026-06-30] | Approved pages locked | `welcome.mdx` and `first-15-minutes.mdx` content is client-approved — do not rewrite it; only additive, content-preserving changes (e.g. adding mockups) are allowed.
- [2026-06-30] | No unrequested execution | Do NOT run agents/builds/passes without explicit user approval. "Add X to the handover" means edit the doc, not do the work. Confirm before dispatching.
- [2026-06-30] | Always use agents | Per CLAUDE.md, ALL file edits, searches, and command execution must go through Task-tool subagents (general-purpose for edits, Explore for search, Bash for commands), NOT direct tool calls from the main thread. The only exception: VIEWING screenshots/images to assess look & feel must happen in the main multimodal thread (subagents return text only). Loop: agent captures screenshot to disk, main thread reads the image.
- [2026-06-30] | Claude UI conversation mockups | ClaudeMessage components with role="claude" have NO bubble background or padding, so placing them as direct children of ClaudeWindow renders the Claude text flush against the window's left edge. Any conversation mockup MUST wrap its ClaudeMessage children in a padding+flex wrapper div: `<div style={{ padding: '20px', display: 'flex', flexDirection: 'column', gap: '20px' }}>` (this is what interface-tour's working conversation mockup uses).
- [2026-06-30] | Sticky sidebar/TOC | A position:sticky child only stays pinned within the HEIGHT of its parent column. DocLayout's `.doc-grid` uses `align-items: flex-start`, which made the side columns only as tall as their content, so the sticky sidebar and TOC scrolled away on long pages. Fix: add `align-self: stretch` to `.doc-sidebar` and `.doc-toc` so each column fills the row height. Do NOT remove `align-items: flex-start` from the row.
- [2026-06-30] | Project Atlas canonical figures | The Project Atlas numbers are LOCKED by `first-15-minutes.mdx`: revenue about $60m (with $ sign), US field-service software for HVAC/plumbing contractors across the US, growth mid-20s percent, gross margin HIGH 70s percent, per-seat subs, NRR above 110 percent. `interface-tour.mdx` had drifted ("Roughly 60m", "low 70s"); always align other pages to the locked first-15 values.
- [2026-06-30] | ClaudeSettings component | New ClaudeUI component `ClaudeSettings.astro` (claude-ui kit) renders a claude.ai Settings panel mockup for the "Instructions for Claude" field (warm cream + clay, never KPMG blue). Use it when illustrating settings screens.
- [2026-06-30] | Page must match its title/scope | The "Thinking & Effort" page originally never explained "thinking" or that it can be turned off. When a page names a concept, it must actually teach that concept.
- [2026-06-30] | Locale switched to US | The site is US English + US dollars. Project Atlas is a US company ($60m revenue, across the US). NEVER reintroduce pounds (£), UK/British geography, or British spellings (organise, summarise, behaviour, judgement, rigour, analyse, maths, adviser, sceptical, etc.). Use US spellings (organize, summarize, behavior, judgment, rigor, analyze, math, advisor, skeptical).
- [2026-07-01] | Astro 6 needs Node 22 in CI | `withastro/action@v3` defaults to Node 20, but Astro 6 requires Node >=22.12.0. The first Pages build failed with EBADENGINE. ALWAYS pass `with: { node-version: 22 }` to the action in `.github/workflows/deploy.yml`.
- [2026-07-01] | Subpath hosting = env-driven base + withBase | The site is served from a GitHub Pages project subpath (`/wiki_claude_PE/`). Base is env-driven in `astro.config.mjs` (`process.env.SITE_BASE ?? '/wiki_claude_PE/'`). EVERY internal link/asset MUST go through `src/utils/withBase.ts` (or `import.meta.env.BASE_URL`) or it 404s under the subpath. Never reintroduce root-absolute internal `href="/..."`, `/favicon.svg`, or `/pagefind/...`. To move to a root host, build with `SITE_BASE=/`.
- [2026-07-01] | Pagefind under a base path | Pagefind assets 404 under a subpath unless: the CSS/JS tags use the base, AND `PagefindUI` is passed `bundlePath: import.meta.env.BASE_URL + 'pagefind/'` (its internal default is `/pagefind/`, which is root-absolute).
- [2026-07-01] | noindex, not robots.txt, on a subpath | The repo is public and content is confidential. A `public/robots.txt` would sit at `/wiki_claude_PE/robots.txt`, which crawlers do not read (they only read the domain-root robots.txt). Use a per-page `<meta name="robots" content="noindex, nofollow" />` in `BaseLayout.astro` instead.
- [2026-07-01] | I push, user deploys | User owns Pages config and deploy triggering. Push code only. Do NOT run `gh workflow run`, `gh api ... /pages`, or change Pages settings in the UI. Reading run status/logs (`gh run list/view/watch`) is allowed for diagnosis.
- [2026-07-02] | Task 10 (critique & elevate pass) completed and marked done in handover with no verification build (user chose "just mark it done") | When the user says a task is simply "done", update the docs to reflect completion only — do NOT run builds, agents, or passes unless they explicitly ask.
- [2026-07-02] | Reserved empty TOC column | .doc-toc was a flex column (flex: 0 0 var(--toc-w)) that reserved 232px at all widths even though the inner .toc content hid at max-width:1240px, squeezing main content and leaving an empty divider strip. DocLayout .doc-grid is now CSS Grid; define grid-template-columns per breakpoint so hidden sidebar/TOC tracks are removed entirely. Do NOT reintroduce a fixed-width flex side column that is only content-hidden.
- [2026-07-02] | Mobile hamburger showed no menu | The inner Sidebar.astro sets .sidebar { display:none } at max-width:1080px; DocLayout's .doc-sidebar.open :global(.sidebar) override must set display: block (not only width) or the opened drawer stays empty. Keep the un-hide fix in the layout open-state override, not in the Sidebar component.
- [2026-07-02] | Grid replaces the align-self:stretch sticky hack | With .doc-grid as CSS Grid, do NOT set align-items on it; the default stretch makes sticky sidebar/TOC columns fill the row height. This supersedes the earlier flex align-self:stretch fix. Do not set align-items:start on the doc grid or sticky columns break.
- [2026-07-02] | Mockups must use container queries, not viewport media | Claude UI mockups (ClaudeWindow, ClaudeComposer) live inside a variable-width article column, so viewport @media is wrong (can render cramped on desktop). Use @container with a container-type: inline-size root and DISTINCT container-name per component (cw, cc) so nested mockups do not query the wrong ancestor.
- [2026-07-02] | Prefer adaptive CSS over magic numbers | User preference: use CSS Grid per-breakpoint templates, container queries, and min()/clamp() so layout adapts on its own. Reserve fixed-pixel breakpoints for genuine UX decisions only (for example, sidebar becomes a drawer).
