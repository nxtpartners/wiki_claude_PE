# Diagram 4: Git save points

Diagram generation prompt (for Gemini or ChatGPT). Output image: `cc-save-points.png`

---

Design a genuinely well made timeline diagram of Git as a simple line of save points for a private equity teaching guide. It shows the reader one thing: every commit is a snapshot you can go back to. This is a client-facing concept graphic for KPMG Private Equity professionals, sitting inside a light, premium, institutional web page. Aim for world-class information design, the kind of clear, reassuring exhibit a senior training deck would put on one page. Calm, senior, precise, and clearly made on purpose.

**The concept to visualize:**

A left-to-right timeline of save points, each one a labeled point on the line with a short note. The reader should understand that each point is a snapshot of the work, that they can always return to an earlier one, and that Claude Code can create these points for them when they ask it to save their work.

- The line runs left to right, with a small sequence of points along it.
- The example points, in order, read: "Start", then "Tidied the data pack", then "Added the checks", then "Fixed the totals".
- The most recent point ("Fixed the totals") is the current one and should be marked as such, for example with a teal accent, so the reader can tell where they are now.

The core idea to land: every commit is a snapshot you can go back to, and you never lose your place.

**Required text labels (use these exactly, correctly spelled):**
- The overall label, verbatim: "Save points (commits)".
- The four example commit notes, in order, verbatim: "Start", "Tidied the data pack", "Added the checks", "Fixed the totals".
- The reassurance, verbatim: "You can go back to any point".
- The action cue, verbatim: "Ask Claude Code to save your work".

**Your creative freedom:**

You own the design. Decide the structure, the composition, the exact form of the timeline, the shape of the points, and the visual metaphor for "save points you can return to." Use a clean horizontal spine with nodes, a subtle track, gentle markers, or any device you judge clearest and most elegant, as long as three things read at a glance: the points sit in order left to right, the latest point is clearly the current one, and the reader can go back to any earlier point. Do not feel bound to any particular arrangement of dots, ticks, connectors, or note placement. Let the timeline feel calm and steady, and let the current point read as the "you are here" marker. Make it beautiful and make it clear.

**Canvas and brand:**
- Light or white canvas (`#FFFFFF` or a very light tone). High resolution. Generous margins and quiet negative space.
- Brand palette, blue-led: deep blue `#00338D`, cobalt `#1E49E2`, sky `#ACEAFF`, teal `#00A3A1`, confidential red `#B42318`, ink `#0B1B33`, muted ink `#5A6678`, hairline `#E3E8EF`, white `#FFFFFF`, soft tint `#F5F7FA`. Lead with blue and ink. Use a teal accent to mark the most recent, current save point; otherwise use teal and red sparingly if at all.
- Typography: Inter or a near-identical clean grotesque. Real, legible, correctly spelled US English. Keep labels short.
- Depth is welcome and tasteful: soft shadow, subtle layering, gentle in-palette gradients, well-made nodes. Never gaudy. Reference bar: world-class information design in the spirit of FT and Bloomberg graphics, Pentagram, and Stripe Press.

**Overall feel:**

A calm, reassuring timeline: ordered, dimensional, and instantly legible, with real craft in every point and label. The reader should instantly understand that every commit is a save point they can go back to, and that Claude Code can create these for them when they ask it to save their work.

**Avoid:** dark or navy backgrounds, neon, purple or violet, sci-fi or circuit-board motifs, hexagon tech patterns, emoji or clip-art stock icons, childish or generic "AI infographic" looks, busy or noisy backgrounds, watermarks, and any misspelled or lorem text.
